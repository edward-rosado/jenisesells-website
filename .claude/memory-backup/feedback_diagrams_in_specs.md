Always include diagrams (ASCII art, flow charts, architecture diagrams) in design specs and documentation produced during brainstorming sessions.

**Why:** Eddie asked for this after a brainstorming session where a middleware data flow diagram was created on request. He found it valuable and wants it as a default, not something he has to ask for.

**How to apply:** During the brainstorming skill's "Present design" and "Write design doc" steps, proactively create diagrams for any data flows, request lifecycles, component relationships, or architecture. Don't wait for the user to ask — include them by default in every spec.
